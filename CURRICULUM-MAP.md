# CKS Curriculum Map

This map follows the CKS curriculum published by CNCF and the Linux Foundation
for Kubernetes v1.34, which was the listed exam version on June 15, 2026.

| Domain | Labs |
|---|---|
| Cluster Setup | 03, 05, 06, 13, 15, 20, 23, 27 |
| Cluster Hardening | 01, 03, 08, 12, 20, 26 |
| System Hardening | 05, 09, 10, 24 |
| Minimize Microservice Vulnerabilities | 04, 09, 10, 11, 12, 13, 19, 24, 27 |
| Supply Chain Security | 06, 07, 16, 21, 22 |
| Monitoring, Logging and Runtime Security | 02, 14, 17, 18, 25 |

## Extra Study

The playlists provide strong task practice, but do not rely on memorizing these
exact scenarios. Make sure you can also:

- Apply current Pod Security Standards without deprecated PSP workflows.
- Use seccomp and AppArmor with the syntax supported by the exam Kubernetes version.
- Explain and implement workload identity and least-privilege service accounts.
- Work with SBOMs, signed artifacts, permitted registries, and static analysis.
- Configure and inspect Kubernetes audit policy and audit logs.
- Write and test Falco rules rather than only reading default alerts.
- Implement pod-to-pod encryption with Cilium or Istio.
- Navigate the allowed documentation quickly under time pressure.
